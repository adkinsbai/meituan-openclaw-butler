# 饭点之外

基于 OpenClaw 的本地生活“全天候私人管家”黑客松项目。

## 比赛背景与项目意义

本次美团黑客松赛道聚焦 **基于 OpenClaw 的本地生活“全天候私人管家”**。这个赛题的关键不只是做一个能聊天的助手，而是探索 Agent 如何真正接入本地生活服务，在合适的时间、地点和场景下帮助用户完成生活决策与服务执行。

美团已经覆盖了外卖、到店餐饮、闪购、电影、酒店、旅游、KTV、按摩足疗、跑腿、买药、洗车养车等大量本地生活服务。但对普通用户来说，美团最高频的使用场景仍然是“饭点点外卖”，其次才是在商场、聚会、旅行或应急时临时想起某些服务。

这意味着一个核心机会：

> 用户很多时候不是不需要美团，而是没有意识到“这个场景也可以用美团解决”。

因此，全天候私人管家不应该只是被动回答“我要吃什么”，而应该具备场景感知、服务唤醒、主动发现和主动撮合能力。它让美团从“用户搜索服务”升级为“Agent 主动理解生活场景并组织本地服务”。

饭点之外正是围绕这个命题设计的方案。

## 项目定位

饭点之外不是外卖助手，也不是独立 App，而是嵌入美团 App 的场景感知 Agent 系统。

它要解决的问题是：

> 用户很多时候不是不需要美团，而是没有意识到“这个场景也可以用美团解决”。

饭点之外会在用户没想到美团、没想好做什么、也没人一起玩的时候，主动发现附近机会，低成本撮合兴趣相近的人，并把兴趣变成平台内可预约、可支付、可履约的真实本地生活场景。

## 核心能力

### 1. 场景感知服务唤醒

用户不需要知道美团有哪些功能，只需要表达生活状态。

例如：

```text
我在商场逛着有点累，不知道接下来干嘛。
```

Agent 会结合位置、时间、天气、同行人和用户偏好，推荐当前最合适的美团服务，例如热饮、电影、甜品团购、晚餐排队等。

### 2. 本地生活发现与可控破圈

当用户说：

```text
我不知道接下来干嘛。
```

Agent 不返回功能列表，而是给出三档行动建议：

- 稳妥：符合历史偏好。
- 轻破圈：和历史兴趣相邻，新鲜但不离谱。
- 大胆尝试：低风险、低成本、可取消的新体验。

### 3. 主动组局

多人使用美团能带来更高消费和更完整的本地生活链路。饭点之外不等用户自己组局，而是主动帮用户把局成掉。

例如：

```text
今晚一个人也不知道玩啥，有没有人一起玩点什么？
```

Agent 可以生成：

```text
4 人轻桌游局
时间：19:30-21:30
地点：岛上桌游望京店
预算：人均 68 元
成局后可接夜宵团购
```

## Agent 架构

系统由两类 Agent 协作：

| Agent | 代表谁 | 职责 |
| --- | --- | --- |
| 个人小 Agent | 用户主人 | 维护兴趣、预算、安全边界和空闲倾向，代表主人匿名表达意向 |
| 平台大 Agent | 美团平台 | 扫描商圈资源、聚类需求、生成候选局、协调商户履约 |

核心原则：

```text
大 Agent 负责发现机会，小 Agent 负责保护主人并判断是否值得参与。
```

## 主动撮合流程

```text
平台大 Agent 扫描商圈活动和商户资源
  ↓
向相关个人小 Agent 询问匿名兴趣摘要
  ↓
个人小 Agent 返回授权范围内的偏好、空闲、预算、探索度
  ↓
平台大 Agent 聚类出潜在活动局
  ↓
平台大 Agent 匹配商户、时间、套餐、人数、预算
  ↓
平台大 Agent 生成候选局提案
  ↓
个人小 Agent 分别评估是否适合自己的主人
  ↓
接近成局时，小 Agent 向主人发起确认
  ↓
主人确认
  ↓
平台内完成预约、买券、AA、提醒和安全留痕
```

## A2A 边界

Agent-to-Agent 可以交换：

- 兴趣标签
- 商圈级位置
- 可用时间窗口
- 预算范围
- 是否接受陌生兴趣局
- 安全偏好

Agent-to-Agent 不允许交换：

- 真实手机号
- 精确实时位置
- 家庭住址
- 支付信息
- 未授权日程详情
- 未授权消费承诺

所有预约、买券、支付、AA 动作都必须由用户最终确认。

## Mock Tool Layer

黑客松阶段使用 Mock CLI 模拟美团服务能力。

核心工具：

```text
life scene.detect
life service.suggest
life activity.discover
life preference.update
life broker.scan
life broker.poll-agents
life broker.propose-groups
life agent.explore
life a2a.propose
life a2a.negotiate
life group.plan
life group.match
life group.invite
life group.reserve
```

当前 Demo 使用 Mock Tool Layer，但 Agent 编排、撮合、确认、履约链路是真实可替换的。未来只需要把 Mock 工具替换成美团内部 API。

## 黑客松 Demo

建议展示三段：

1. **个人发现**
   用户输入“我不知道接下来干嘛”，Agent 根据位置、时间、天气推荐热饮、电影、手作体验。

2. **平台主动撮合**
   平台大 Agent 扫描望京商圈，发现桌游店有空位，多个小 Agent 表示主人喜欢桌游/密室/轻社交，生成 4 人桌游局。

3. **成局履约**
   用户确认后，系统发起邀请、收集 RSVP、锁定商户、生成 AA，并推荐后续夜宵团购。
